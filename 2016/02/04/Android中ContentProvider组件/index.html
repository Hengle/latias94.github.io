Android ContentProvider组件总结

发表于 2016-02-04   |   分类于 Android   |     |   阅读次数

ContentProvider共享数据

在Android中，每个应用程序的数据都是私有的，不管是文件数据、数据库数据还是其它类型的数据都不能被其它程序访问。为了解决这个问题，Android提供了一个ContentProvider类用于实现多应用程序间数据的共享，ContentProvider以类似数据库中表的方式将数据暴露，外部程序采用URI来访问该“数据库”，Content Provider定义了一组用于提供其他应用程序存取数据的标准方法。 应用程序可以在Content Provider中执行 查询、修改、添加、删除数据的操作。

Android提供了一些已经在系统中实现的标准Content Provider，比如联系人信息，图片库等等，你可以用这些Content Provider来访问设备上存储的联系人信息，图片等。

我们可在android.provider包下面找到一些Android提供的ContentProvider。通过获得这些ContentProvider可以查询它们包含的数据，当然前提是已获得适当的读取权限。

public boolean onCreate()
ContentProvider创建后被调用，Android开机后，ContentProvider在其它应用第一次访问它时才会被创建。

public Uri insert(Uri uri, ContentValues values)
该方法用于往 ContentProvider 添加数据。

public int delete(Uri uri, String selection, String[] selectionArgs)
该方法用于从 ContentProvider 删除数据。

public int update(Uri uri, ContentValues values, String selection, String[] selectionArgs)
该方法用于更新 ContentProvider 中的数据。

public Cursor query(Uri uri, String[] projection, String selection, String[] selectionArgs, String sortOrder)
该方法用于从 ContentProvider 中获取数据。

public String getType(Uri uri)：返回当前Url所代表数据的MIME类型
数据属于集合类型，MIME类型字符串以vnd.android.cursor.dir/开头，
数据属于非集合类型数据，MIME类型字符串以vnd.android.cursor.item/开头

ContentValues的简单介绍

ContentValues 和 HashTable 类似都是一种存储的机制，但是两者最大的区别就在于：ContentValues 只能存储基本类型的数据，像string、int之类的，不能存储对象这种东西，而HashTable却可以存储对象。

在SDK中，ContentValues的介绍为：

This class is used to store a set of values that the ContentResolver can process.
就是用于保存一些数据（string boolean byte double float int long short …）信息，这些信息可以被数据库操作时使用。

  • ContentValues() Creates an empty set of values using the default initial size
  • ContentValues(int size) Creates an empty set of values using the given initial size
  • ContentValues(ContentValues from) Creates a set of values copied from the given set

ContentResolver

简介

当外部应用需要对 ContentProvider 中的数据进行添加、删除、修改和查询操作时，可以使用ContentResolver 类来完成，要获取 ContentResolver 对象，可以使用 Activity 提供的getContentResolver() 方法。

方法

ContentResolver 类提供了与 ContentProvider 类相同名字的四个方法：
public Uri insert(Uri uri, ContentValues values)
该方法用于往 ContentProvider 添加数据。

public int delete(Uri uri, String selection, String[] selectionArgs)
该方法用于从 ContentProvider 删除数据。

public int update(Uri uri, ContentValues values, String selection, String[] selectionArgs)
该方法用于更新 ContentProvider 中的数据。

public Cursor query(Uri uri, String[] projection, String selection, String[] selectionArgs, String sortOrder)
该方法用于从 ContentProvider 中获取数据。

  • projection表示要查询哪些列，比如查询媒体库，可能要关注音乐的艺术家，长度，文件位置等。
  • selection表示查询条件，就是查询里面的where语句，selectionArgs是查询条件的值。
  • sortOrder是排序方式。

这些方法的第一个参数为Uri，代表要操作的是哪个 ContentProvider 和对其中的什么数据进行操作，假设给定的是： Uri.parse(“content:/latias.github.io/person/10”)，那么将会对主机名为latias.github.io的 ContentProvider 进行操作，操作的数据为person表中id为10的记录。

ContentUris

ContentUris是content URI的一个辅助类，用于获取Uri路径后面的ID部分。它有两个方法很有用，具体如下所示:

public static Uri withAppendedId(Uri contentUri, long id)，这个方法负责把id和contentUri连接成一个新的Uri。比如在我们这个例子当中是这么使用 的：ContentUris.withAppendedId(Diary. DiaryColumns.CONTENT_URI, rowId)。如果rowId为100的话，那么现在的这个Uri的内容就是：
content://com.ex09_2_contentprovider.diarycontentprovider/diaries/100。

public static long parseId(Uri contentUri)，这个方法负责把 content URI 后边的id解析出来，比如现在这个content URI 是content://com.ex09_2_contentprovider.diarycontentprovider/diaries/100，那么这个函数的返回值就是100。

监听ContentProvider中数据的变化

如果ContentProvider的访问者需要知道ContentProvider中的数据发生变化，可以在ContentProvider发生数据变 化时调用getContentResolver().notifyChange(uri, null)来通知注册在此URI上的访问者

public class PersonContentProvider extends ContentProvider {
    public Uri insert(Uri uri, ContentValues values) {
        db.insert("person", "personid", values);
        getContext().getContentResolver().notifyChange(uri, null);
    }
}

Uri

URI（统一资源标志符）表示要操作的数据，比如：查询日程表，查询联系人，查询媒体库等，一个Uri由以下几部分组成：

A：ContentProvider（内容提供者）的scheme已经由Android所规定， scheme为：content://

B：主机名（或叫Authority）用于唯一标识ContentProvider，外部程序根据该标识来找到它，与AndroidMainfest.xml中注册的provider的android:authorities属性对应，一般采用程序“包.类”全小写来表示。
C&D：路径（path）可以用来表示我们要操作的数据，路径的构建应根据业务而定，如下:

  • 操作person表中id为10的记录，可以构建这样的路径: /person/10
  • 操作person表中id为10的记录的name字段，person/10/name
  • 操作person表中的所有记录，可以构建这样的路径: /person
  • 操作的数据也可以是文件、xml或网络等其他存储方式，把字符串转换成Uri，可以使用Uri类中的parse()方法，如下：

Uri uri = Uri.parse("content://com.ljq.provider.personprovider/person")

UriMatcher

因为Uri表示要操作的数据，所以我们经常需要解析Uri，并从Uri中获取数据。Android系统提供了两个用于操作Uri的工具类，分别为UriMatcher和ContentUris ，UriMatcher类用于匹配Uri，用法如下：
首先第一步把你需要匹配Uri路径全部给注册上，如下：

//常量UriMatcher.NO_MATCH表示不匹配任何路径的返回码
UriMatcher sMatcher = new UriMatcher(UriMatcher.NO_MATCH);
//如果match()方法匹配content://com.oracle.personprovider/person路径，返回匹配码为1
sMatcher.addURI("com.oracle.personprovider", "person", 1);//添加需要匹配uri，如果匹配就会返回匹配码
//如果match()方法匹配content://com.oracle.personprovider/person/230路径，返回匹配码为2
sMatcher.addURI("com.oracle.personprovider", "person/#", 2);//#号为通配符

注册完需要匹配的Uri后，就可以使用sMatcher.match(uri)方法对输入的Uri进行匹配，如果匹配就返回匹配码，匹配码是调用 addURI()方法传入的第三个参数，假设匹配。代码如下：

switch (sMatcher.match(Uri.parse("content://com.oracle.personprovider/person/10"))) { 
   case 1
     break;
   case 2
     break;
   default://不匹配
     break;
}

ContentUris

ContentUris类用于操作Uri路径中的ID部分，常用方法如下：
withAppendedId(uri, id)用于为路径加上ID部分：
Uri uri = Uri.parse("content://com.oracle.personprovider/person");
Uri resultUri = ContentUris.withAppendedId(uri, 10);
//生成后的Uri为：content://com.oracle.personprovider/person/10

parseId(uri)方法用于从路径中获取ID部分：
Uri uri = Uri.parse("content://com.oracle.personprovider/person/10");
long personid = ContentUris.parseId(uri);//获取的结果为:10
